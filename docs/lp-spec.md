# LP仕様（トップページ index.html のホームビュー）

1. **ヒーロー**: h1（緑・900・字間-0.5px・3行）「麻雀のスコア記録、<br>まだ電卓と紙とペンで<br>やってますか？」
   → SVGイラスト（過去と現在の対比） → 説明文 → CTA → バッジ3つ → 「最近の試合」バナー → 「これまでの試合を振り返る ›」
2. 3ステップ「必要なものはスマホのみ。」（薄青）
3. 比較表「手書きのスコア表は、もう卒業。」（薄緑）
4. 特徴カルーセル「スマホ1台で、ここまでできる。」（白）
5. クロージング「麻雀のスコア記録は、まじゃすこで。」（薄緑）
6. home-content: #howto 使い方 → **Safariお気に入り登録の案内**（.bookmark-howto） → lp_bottom広告枠
7. フッター（緑・2列リンク: score-basics.html/faq.html/ブログ/お知らせ/RSS/規約/PP/X/問い合わせ。
   **「使い方」リンクは置かない**＝ヘッダー右上の「使い方」と重複するため。blog-site.jsのrenderChromeも同じ）

## 必須文言・ルール
- ファーストビュー説明文:「「まじゃすこ」は、表示点数を入力するだけでウマ・オカ込みのスコアを自動計算して、URLひとつで全員のスマホに共有できる無料のサービスです。」
- **各セクションは「タイトル→表示→締め」の3段構成**。締め（.start-note）: 3ステップ=「たった10秒で、次の対局をスタート。」／比較表=「めんどうな計算は、ぜんぶ まじゃすこへ。」（半角スペース）／特徴=「そしてこの全部が、無料。」
- CTAはオレンジ・影なし・横長ピル（max-width:380px）。**CTA下はバッジ3つ**（.close-badges: 完全無料/アプリDL不要/会員登録不要。ヒーローとクロージングで共通）
- 見出しは22px/900統一。解説系本文は15px（feature pのみ14.5px）。行間1.8前後
- コピーは行動経済学（損失回避・アンカリング等）を意識。Walica風フラットデザイン参考（模倣はしない）

## ヒーローのSVGイラスト
- 左（薄い=過去 opacity0.55）: 電卓・くしゃくしゃの紙・鉛筆（**消しゴム・金具なし**）がゴミ箱へ。鉛筆は**くしゃくしゃの紙に重なる位置・過去グループ内の最前面**（電卓・紙より後に描く）
- 中央: 点線の乗り換え矢印（進行方向に沿った三角の矢じり）
- 右（明るい=現在）: iPhone風スマホのスコア収支画面＋淡い光の円＋✨型スパークル（＋印は使わない）

## 特徴カルーセル
- 通常の横スクロール＋スナップ（無限ループは廃止。`initFeaturesCarousel`）
- カード幅モバイル62%（次のカードが半分見える）・snap-align:start・700px以上は300px＋左右矢印
- UX: 両端28px白フェード＋現在位置ドット.features-dots（rAF不使用）
- 9項目（タイトル1行8〜10字・16px・本文2文）: スコアを一瞬で計算/全員のスマホに共有/推移グラフも自動/最大40人で回し打ち/チーム戦にも対応/ルールは自由自在/チップや焼き鳥も記録/成績表も自動で作成/CSVで書き出せる
- アイコンは絵文字ではなくインラインSVG（フラットトーン: #177083・#9fc3bd・#E4F3EC・差し色#E75620）。背景色fi-blue/fi-greenは交互
- **9枚すべてにオレンジ#E75620のワンポイントを必ず1つ入れる**（電卓=キー1個/共有=電波/グラフ=終点●/40人=✨/トロフィー=エンブレム/スライダー=つまみ1個/チップ=天面リング/成績表=順位バッジ/CSV=ダウンロードバッジ）

## クロージング
- ミニイラスト（スマホ＋✨SVG .close-illust）→ h2 → ひとこと2行 → バッジ3つ → CTA

## 使い方セクション（#howto）
- 「1分でわかる「まじゃすこ」の使い方」= Walica風4ステップ。**カードにしないシンプルな置き方**
- 見出し22px/900・ステップh3 20px/900・説明文15px。イントロは「会員登録もインストールも不要。」で改行
- 各ステップ: 中央見出し → 緑ブロック.howto-shotにスマホ画面SVG → 短い説明文
- スマホSVG: viewBox 220×400・比率約1:1.94・ベゼル8px均一・ダイナミックアイランド。**ラベルは「試合形式」**（対局形式/対戦形式とは書かない）
- ステップ1のグループ名例は「まじゃすこ学園サッカー部」（日付は使わない）

## Safariお気に入り登録の案内（.bookmark-howto・使い方の直後）
- 見出し「番外編：Safariからすぐ開く方法」（22px/900=使い方と同じ。モバイル375pxで1行に収まる長さを保つ）＋サブ15px3行
  「Safariの“お気に入り”に一度登録しておけば、<br>次の対局からはワンタップで「まじゃすこ」へ。<br>ホーム画面のアプリのような使い心地になります。」
- 薄緑 #E4F3EC の**1枚のブロック（.bm-wrap）が3画面全体の背景**。スマホSVG3枚＋間にシェブロン矢印（#9fbcb7・2枚重ね）
- 押す順に**オレンジ丸＋白数字のバッジ1〜5**（1=Safariの「…」、**2=「ブックマークの追加先」**、3=お気に入り、4=保存、5=お気に入りのMタイル）。
  **手順の説明はこの画像内バッジのみ**（外側に説明文・凡例は置かない）。1画面目のメニュー塊はURLバーのすぐ上に配置
- スマホの中身は実際のiOS画面を忠実にデフォルメ（Safariメニュー・ブックマーク追加シート・お気に入りグリッド。
  他サイトのタイルはYouTube風▶/X風/カメラ/トークの汎用アイコン、Mタイルはファビコンと同じティール枠+白+オレンジM）
- レスポンシブ: **常に3画面を横並び**にし、モバイルではSVGごと等倍縮小（flex:1・width:100%）してスクロールなしで全体が見える。
  700px以上は矢印が大きくなるだけ（26px、未満は14px）
- 末尾の注記（.bm-note・**左詰め**）「※iPhone（Safari）の場合。Androidはブラウザのメニュー →「ブックマーク」から追加できます。」
  （.bookmark-howto > p のセンター指定に勝つよう .bookmark-howto .bm-note の2クラスで指定）

# SNS運用（X・Instagram）

## アカウント
- X: **@majasco_jp**（プロフィール文・固定ポスト設定済み。twitter:site全ページ設置済み）
- Instagram: まじゃすこ公式（名前欄「まじゃすこ｜麻雀スコア記録アプリ」・ウェブサイト欄 https://majasco.jp）
- **投稿の実行は必ずユーザーが手動**。Claudeは画像と文面を用意して渡すだけ

## 共通ルール
- 用語ルール（CLAUDE.md）を投稿文・画像内文言にも適用（金銭語NG・「スコア記録」・bonus）
- 画像はブランドトーンで生成（docs/image-tools.md）。**画像内のイラストに絵文字は使わない**（キャプション内の絵文字はOK。画像内のメダル🥇等アプリ実画面の再現はOK）
- URLの扱い: **Xは本文の最後に生URL**（OGPカードが出る）。**Instagramは本文リンク不可**→「プロフィールのリンクから」誘導
- ハッシュタグ（IG基本セット・10個前後）: #麻雀 #麻雀好きと繋がりたい #麻雀好き #セット麻雀 #麻雀アプリ #三麻 #四麻 #麻雀初心者 #麻雀サークル #まじゃすこ（Xは0〜2個まで）

## 投稿タイプ
1. **機能紹介**（単発画像＋2〜3文＋CTA）— tools/make_ig_series.py の feature-1〜6
2. **使い方カルーセル**（IG6枚組）— howto-1〜6
3. **アップデート告知** — /news/記事公開 → X → IG の順で連動
4. **記事シェア**（X向け: 【ブログ更新】＋タイトル＋ひとこと要約＋URL）

## 制作手順（画像つき投稿）
1. `tools/` のスクリプトで生成（新パターンは既存スクリプトをコピーして作る）
2. 出力先は `まじゃすこ素材/`（コミットされない）。IGシリーズは `まじゃすこ素材/ig/`
3. **生成したらClaude自身がReadで画像を開き、下のQAチェックリストを通してから**ユーザーに提示
4. 投稿文はこのdocのテンプレ・過去文面（git履歴/会話）を踏襲

## 投稿前QAチェックリスト
- [ ] 画像内の文字がスマホ枠・カード枠・画面の内側に収まっている
- [ ] 装飾（✨・ドット）が文字と重なっていない
- [ ] 用語ルール違反がない
- [ ] majasco.jp の表記がある
- [ ] サイズ: IG=1080×1080 ／ Xヘッダー=1500×500（左下アイコン・上下60px安全域）／ OGP=2:1トリミング考慮
- [ ] 文面: Xは末尾にURL／IGはプロフィール誘導・ハッシュタグ

## 運用サイクル
- 新機能リリース時: /news/記事 → X告知 → IG告知（同素材使い回しOK）
- 平常時: 機能紹介シリーズを2〜3日おきに1本
- 反応の良し悪し・トラブル・学びは下の「落とし穴」へ追記して蓄積する

## 落とし穴（トラブルの記憶。修正したらここに追記）
- **Xのカードキャッシュ**: 同一URLは約1週間キャッシュ。テスト投稿は `?v=N` を変える。OGP画像の差し替えは**ファイル名ごと変更**（ogp.png→ogp2.png方式）＋全ページのmeta書き換え
- **Xカードにタイトル文は出ない**（画像＋ドメインのみ。2023年仕様）
- **画像内文字のはみ出し**: PILは自動折返ししない。2026.07.11にhowtoカルーセルで発生（チップ幅・チャット文・合計バー）→ 生成後の目視QAを必須化
- **カラー絵文字の描画**: seguiemj.ttf + `embedded_color=True` でPILから描画可能（🥇🥈🥉実績）
- **装飾の重なり**: スパークルは文字の行末付近（右端）に置かない

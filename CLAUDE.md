麻雀のスコア集計・共有ができる無料のWebアプリ。ビルド不要の静的サイト（Vanilla JS + Firebase Realtime Database）。

## 用語ルール（AdSense対策・超重要）
- 「円」「金額」「精算」「賭け」など現実の金銭を連想させる語は禁止
- **「倍率」「収支」は使用OK**（2026.07.10ユーザー確認済み。ギャンブル語彙には該当しない扱い）
- **統一用語（2026.07.10「収支」モデルで確定。UI・コード・記事すべてこれに従う）**:
  - 「スコア倍率」（旧: スコア係数/レート。ptに掛ける倍率。保存キーは互換性のため `settings.rate` のまま変更しない）
  - 「チップ倍率」（チップ差に掛ける倍率。保存キー `settings.chipRate`）
  - 「ゲーム開始時チップ数」（保存キー `settings.startChips`。未設定は0）
  - 「チップ差」＝ 終了時チップ数 − 開始時チップ数（旧: ボーナス枚数。試合ごとの保存キーは `round.chips` のまま）
  - 計算後の値は「スコア収支」（pts×スコア倍率）「チップ収支」（チップ差×チップ倍率）「合計収支」（両者の合計）
  - **収支の単位は「bonus」**（2026.07.10確定）。「スコア収支」「チップ収支」は値の名称であって単位ではない。
    数値には小さく `bonus` を付けて表示（例: `+7,000bonus`）。
    倍率の例示は「チップ倍率が500倍なら、チップ1枚 → 500bonus」「スコア倍率が10倍なら、1.0pt → 10bonus」の形式で書く
  - 「レート」「係数」「チップ換算」「〜ボーナス」という表記は使わない（換算表示の呼び名は「収支換算」、集計カードは「収支集計」）
- 麻雀の「点数計算（符・翻から1局のアガり点を出す、手役の計算）」と
  「スコア計算/集計（対局後に持ち点→返し点・ウマ・オカを反映して順位・ptを出す）」は別物。
  **このアプリが解決するのは後者のみ**。前者（符・翻・役の計算）はアプリの対象外で、ユーザー自身が数える前提。
  - NG例: 「点数計算がめんどうだからアプリで解決」「点数計算の基本（初心者向け）」という見出しでウマ・オカ・返し点を説明する
    → 中身がウマ・オカ・返し点の話なら、見出しは必ず「スコア計算」にする（2026.07に index.html の見出しで実際に混同があり修正した前例あり）
  - OK例: 「符・翻の点数計算は自分で。半荘のスコア集計は、まじゃすこで。」のように明確に切り分けて併記するのはよい
  - 符・翻・役・点数表を解説する記事（blog-score-table.html, blog-fu-han.html, blog-yaku-list.html等）はテーマとして「点数計算」を扱ってよいが、
    本文中でアプリの守備範囲を説明する箇所は必ず「半荘の集計」「スコア計算」という語に切り替えること

## 機能アップデートのロードマップ（2026.07.10開始）
軽量・高速なURL共有モデルのUXを維持したまま、以下を段階的に実装する。

### フェーズ1: 用語整理と対戦管理の基盤 【完了 2026.07.10】
1. 用語の一括置換（→上の「用語ルール」参照）
2. メンバーリストの可変長化（1グループ最大40名=MAX_MEMBERS。各試合はその中からnumPlayers人を選んで対局。
   メンバー4人以上のときは設定画面で4人麻雀/3人麻雀を選択可＝回し打ち対応。3人ちょうどなら3人麻雀に自動固定）
3. スコア入力画面: メンバー数がnumPlayersより多いグループのみ上部に「今回の対局メンバー」選択チップを表示
   → ちょうどのグループは従来通り選択なしで入力（後方互換UX）。デフォルト選択は直前の試合のメンバー
4. 設定変更モーダルでメンバー追加・削除可（対局済みメンバーは削除不可。削除時はrounds[].membersのindexを旧→新に振り直す）
5. 実装の要点: 各試合は `round.members`（playerNamesへのindex配列）を持つ。**旧データ互換は `roundMembers(r, s)` に集約**
   （members無し→先頭numPlayers人とみなす）。集計は `gameTotals()` / `playedFlags()` / `rankCountHtml(rounds, s)` が参加試合ベースで計算。
   未参加メンバーはスコアカードで「未参加」表示・対局履歴の表で「—」・グラフから除外

### フェーズ2: 収支計算ロジック 【完了 2026.07.10（同日中に「収支」モデルへ改訂済み）】
計算仕様: スコア収支 = pts × スコア倍率 ／ チップ収支 = チップ差 × チップ倍率 ／ 合計収支 = 両者の和。
チップ差 = ゲーム終了時チップ数 − ゲーム開始時チップ数（`settings.startChips`）。
1. ゲーム設定（セットアップ+設定変更モーダル）に「収支オプション」:
   「チップの収支計算を有効にする」チェック（ONでチップ倍率＋ゲーム開始時チップ数欄を表示）＋「焼き鳥🐔を記録する」チェック＋注釈（例: 100.0pts × スコア倍率10倍 → 1000スコア収支）
2. スコア入力シート: 収支計算ONのグループのみ、各メンバーの点数行の下に「終了時チップ」「チップ差」の**連動入力**サブ行（`.sheet-subrow`）と🐔トグルを表示。
   どちらを入力しても他方が自動算出される。基準値 = `chipBaseline(m)` = 開始時チップ数 + それまでの試合のチップ差累計（修正時は対象試合より前まで）。
   終了時チップ欄のplaceholderに現在の基準値を表示。値はメンバーindexキーで保持（sheetChips/sheetYaki）し、対局メンバー切替でも消えない
3. ゲーム画面・結果詳細に「収支集計」カード（2026.07.10にメリハリ重視で再設計）:
   **主役＝メンバーごとの合計収支を大きなミニカード（`.bonus-grid`/`.bonus-cell`、2列グリッド・値の大きい順・bonus単位付き）**で常時表示。
   内訳（スコア合計/スコア収支/チップ差/チップ数/チップ収支/合計収支/焼き鳥）は小さめグレーの表（`.bonus-detail-table`）を「内訳を表示」タップで展開。計算は `bonusTableHtml(g)` に一元化
4. 保存キー（**コード上のキー名・関数名は互換のため bonus 系のまま**）: `settings.bonusEnabled`（bool）/ `settings.chipRate`（数値・未設定は1倍扱い）/ `settings.startChips`（数値・未設定は0）/ `settings.yakitori`（bool）、
   試合ごとに `round.chips`（**チップ差**の配列・scoresと同じslot順）/ `round.yakitori`（0/1配列）。**旧データはキー無し=無効/0枚として一切影響なし**
5. 対局履歴の表は記録がある試合のみセル内に「+3枚」「🐔」を小さく表示。スコアカードの換算トグルは「収支換算」（スコア収支を表示）

### フェーズ3: チーム戦 【完了 2026.07.10】
- 保存キー: `settings.teamMode`（bool）/ `settings.teams` = `[{ name, members: [playerNamesへのindex...] }]`。
  チーム順位は所属メンバーの合計pts。収支計算ONならチーム合計収支（メンバーごとの丸め済み収支の合算）も表示
- セットアップ: メンバー4人以上で「チーム戦（オプション）」カードを表示。ONでチーム編成エディタ
- チーム編成エディタ（`teamEditorCtx` でセットアップ/設定変更モーダル共用。関数: renderTeamEditor/moveTeamMember/addTeam/removeTeam/randomizeTeams/syncTeamsWithIds）:
  - メンバーチップの**ドラッグ＆ドロップ**（HTML5 DnD）＋**タップで次のチームへ移動**（スマホ用代替）
  - チーム追加（3チーム以上可）・削除（最低2チーム、削除チームのメンバーは先頭チームへ）・🎲ランダム編成（シャッフル後に均等配分）
  - チーム名は編集可（空ならチームA/B/...）。**再初期化前に必ずcaptureTeamNames()**（入力中の名前がrenderで消えるバグを修正済み）
- メンバーとの同期: `syncTeamsWithIds` が「存在しないid除去＋未所属idを人数最少チームへ」を一元処理。
  設定変更モーダルでのメンバー削除時はチームのid（位置）を詰めて振り直す
- 表示: ゲーム画面・結果詳細の最上部に「チーム成績」カード（メダル順位・メンバー名・pts・合計収支）。設定カードに「チーム戦: 赤組（…） vs 白組（…）」
- チーム戦OFFにしても`settings.teams`は保持し、再度ONで編成を復元
- 旧データ互換: teamMode/teamsが無いグループは一切表示・計算に影響なし

- **チップ設定**: チップ倍率はスコア倍率と同じセレクト＋カスタム形式（`CHIP_RATE_PRESETS`、デフォルト×500）。ゲーム開始時チップ数の初期値は20
- **登録名管理モーダル**: 各名前に[ゲームに追加]ボタン（追加済みはラベル表示）。モバイルでdatalistを使わず直接追加できる

- **チーム識別**: 色ドット廃止→記号 `TEAM_MARKS = ['◯','△','◻','✕','◇','☆']`（`teamMarkOf`/`teamMarkHtml`）。ヘッダー背景色と被らない
- **用語**: ゲーム画面のカード「合計スコア」→「スコア収支」に改称。収支内訳の行は「スコア（pts）」

- **個人成績は転置**（`rankCountHtml`）: 行=メンバー、列=1着..n着/試合数/平均着順/ラス回避。
  名前列は `.stats-table .sticky-col`（position:sticky）で横スクロール時も固定。メンバーが増えても縦に伸びるだけ
- **設定変更モーダルの妥当性表示**: 対局形式は変更不可の明記／ルール変更で過去試合が再計算される⚠警告／
  削除不可メンバーは「対局済み」「最低N人」ラベル（タップで理由をトースト表示）／チーム戦4人未満は保存ブロック／
  収支計算OFF中に試合を修正しても記録済みチップ・🐔は引き継ぐ（submitRoundでメンバー対応再配置）

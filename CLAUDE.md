# まじゃすこ / majasco — 開発ガイド（コア）

麻雀のスコア記録・共有ができる無料Webアプリ。ビルド不要の静的サイト（Vanilla JS + Firebase Realtime Database + Firebase Hosting）。

- 本番: https://majasco.jp ／ リポジトリ: https://github.com/1046nee/mahjong-score（ブランチ: main）
- mainへpush → GitHub Actionsが自動デプロイ（追加作業不要）
- X公式: @majasco_jp ／ Instagram公式あり（詳細は docs/sns-ops.md）

## 絶対ルール（全タスク共通）
1. **作業前に必ず `git pull origin main`**（ユーザーがGitHub Web UIで直接編集することがある。巻き戻し厳禁）
2. 変更後は**ブラウザ実機で検証してから完了報告**（mcp__Claude_Browser__*。スクリーンショットはタイムアウトしやすいのでDOM評価を主体に。検証で作ったFirebaseセッションは `db.ref('sessions/ID').remove()` で削除）
3. **計算ロジックを触ったら /tests.html がALL PASSであることを確認**してからコミット
4. 変更ごとに `git add → commit（日本語で内容と理由）→ push origin main` まで行う
5. **データ互換性が最優先**: 旧データはキー無し=無効扱いで計算に影響させない。保存キー名は変えない（settings.rate等bonus系のまま）
6. 大きめの置換はscratchpadにPythonスクリプトを書いて実行する（アンカー文字列をassertで検証してから置換。ヒアドキュメント直書きはエスケープ事故のもと）
7. **トラブルを修正したら、該当docの「## 落とし穴」に1行追記する**（同じ失敗を繰り返さないための記憶。新しい学びも同様）
8. 仕様を変えたら該当docの記述も更新する（docには「今の正」だけを書く。経緯はgit log）。このCLAUDE.mdは薄く保つ

## 用語ルール（AdSense対策。UI・記事・SNSすべての文言で厳守）
- 「円」「金額」「精算」「賭け」「レート」「係数」など現実の金銭を連想させる語は**禁止**。「倍率」「収支」はOK
- 提供価値は**「スコア記録」**と呼ぶ（行為の説明として「スコアを自動計算」はOK）
- 収支の単位は**bonus**（例: +7,000bonus）。スコア収支=pts×スコア倍率／チップ収支=チップ差×チップ倍率／合計収支=和。
  チップ差=終了時チップ−開始時チップ（持ちチップ）
- **「三麻」「四麻」**表記（初出やSEOタイトルは「三麻（3人麻雀）」併記可）。アプリ内の履歴は「過去の試合」
- 符・翻・役の**「点数計算」はこのアプリの対象外**。アプリが解決するのは対局後の「スコア計算/集計」。
  見出し・説明での混同禁止（詳しい例: docs/blog-spec.md）
